Gumamela Daycare Management System

Theoretical Framework and Conceptual Model

Theoretical Foundation

The Gumamela Daycare Management System is built upon several theoretical frameworks that guide its design and implementation:

1. Social Learning Theory (Bandura, 1977)

Informs the progress tracking and reporting system, emphasizing the importance of observation and modeling in child development.

2. Ecological Systems Theory (Bronfenbrenner, 1979)

Shapes the holistic approach to child development, considering multiple environmental systems that influence a child's growth.

3. Information Systems Success Model (DeLone & McLean, 2003)

Guides the system's design to ensure information quality, system quality, and service quality for optimal user satisfaction.

System Architecture

The system follows a three-tier architecture:

1. Presentation Layer

Responsive web interface built with HTML5, CSS3, and JavaScript, ensuring accessibility across devices.

2. Application Layer

PHP-based backend handling business logic, user authentication, and data processing.

3. Data Layer

MySQL database for secure and efficient data storage and retrieval.

4. Security Framework

Implements role-based access control (RBAC), data encryption, and secure authentication mechanisms.

Conceptual Model

The system's design is based on the following conceptual model that illustrates the relationships between key entities:

Users (Admin/Teacher/Parent) interact with the system to manage Students, track Progress, and participate in Events.

Progress Reports document student development across multiple domains.

Attendance Records track student participation and engagement.

System Settings control application behavior and configurations.

Key Relationships:

  • One Parent can have multiple Students
  • One Teacher can manage multiple Students
  • Each Student can have multiple Progress Reports
  • Multiple Students can be registered for multiple Events
  • Attendance is recorded per Student per day

Performance Metrics

The system includes built-in metrics to evaluate its effectiveness:

1. User Engagement

Track login frequency, feature usage, and session duration.

2. Academic Progress

Monitor student development across different domains over time.

3. System Performance

Response times, error rates, and uptime statistics.

4. User Satisfaction

Periodic feedback collection from all user roles.
