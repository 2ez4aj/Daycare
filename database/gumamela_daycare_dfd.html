üìã DFD Analysis

The Data Flow Diagrams show how data moves through the Gumamela Daycare Management System, from external entities through various processes to data storage and back to users.

üìä Level 0 - Context Diagram

  • Shows the system as a single process
  • Two external entities: Parent and Admin/Teacher
  • One central data store: Daycare Database
  • Data flows represent major interactions
  • High-level view of system boundaries

üîç Level 1 - Process Decomposition

  • Breaks down the main system into 10 processes
  • Shows detailed data stores (D1-D10)
  • Illustrates inter-process communication
  • Reveals data transformation points
  • Maps to actual system functionality

üë• External Entities

  • Parent: Enrolls children, views progress, communicates
  • Admin/Teacher: Manages system, tracks attendance, creates reports
  • Both entities interact with multiple processes
  • Role-based access controls data flow

üíæ Data Stores

  • D1-D6: Core operational data
  • D7-D10: Content and file management
  • Each store corresponds to database tables
  • Supports CRUD operations
  • Maintains data integrity and relationships
